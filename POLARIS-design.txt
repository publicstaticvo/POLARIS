POLARIS IRT Core                  ：大语言模型自适应能力
评测平台
 定位：基于 IRT+CAT 的自动化 LLM 能力评测平台
 核心承诺：提交模型 → 自动评测 → 实时更新排行榜，全程无人工干预
 口号：Submit Once, Ranked by Science


1.   产品核心逻辑
1.1   一句话描述
用户提交模型 API，平台自动运行 IRT 自适应测试，输出带置信区间的能力参数 θ，实时更新全球排行
榜。

3.3   关键技术模块
自动答案解析（Answer Parser）
不同题型用不同解析策略，支持全自动评分：
 class AnswerParser:
          def grade(self, response: str, item: Item) -> float:
             match item.answer_type:
                 case "multiple_choice":
                     #   提取   A/B/C/D，精确匹配
                     return self._extract_choice(response) == item.answer
                 case "numeric":
                     #   提取数字，允许等价形式（0.5             ）
                                                == 1/2
                     return self._extract_number(response) ≈ item.answer
                 case "short_answer":
                     # GPT-4o-mini      作为裁判（judge），批量低成本
                     return self._llm_judge(response, item.answer, item.rubric)
                 case "code":
                     #   执行代码，验证输出
                     return self._execute_and_verify(response, item.test_cases)

async def publish_result(model_id: str, result: EvalResult):
     # 1.   写入数据库
     await db.upsert_model_score(model_id, result)


     # 2.   重新计算排名（增量更新，不全量重排）
     affected_rank_range = await leaderboard.update_rank(model_id, result.theta)


     # 3. WebSocket   推送给所有在线用户
     await ws_manager.broadcast({
          "event": "leaderboard_updated",
          "model": model_id,
          "theta": result.theta,
          "se": result.se,
          "rank": affected_rank_range
     })


     # 4.   通知提交者
     await email.send_completion(model_id, result)
曝光控制（题库安全）
 class ExposureTracker:
        跨所有评测任务追踪题目使用次数"""
      """


      def get_available_items(self, pool: ItemPool,
                                 selected: Set[int]) -> List[Item]:
            available = []
            for item in pool:
                 if item.id in selected:
                      continue
                 #   曝光上限：单题最多被     20%    的模型使用
                 if self.counts[item.id] >= self.e_max:
                      continue
                 available.append(item)


            #   若全部达到上限（极端情况），临时忽略限制并告警
            if not available:
                 logger.warning("All items at exposure limit, ignoring constraint")
                 available = [i for i in pool if i.id not in selected]


            return available

5.1      结果可信的五道防线
 防线1：统计门槛
               时继续选题，不提前结束
     → SE > 0.25
     →   发布结果时标注"评测充分度"（题量 + SE）
 防线2：题库安全
  → MFI+ 曝光控制（单题最大曝光率 < 20%）
  → 泄露检测：正确率显著高于 IRT 预测值时触发警告
  → 每季度滚动替换 15-20% 题目


 防线3：量尺稳定
  → 每领域 10 道锚题，跨批次量尺校正
  → 历史 θ 值不会因新模型加入而改变


 防线4：答案评分
  → 选择题/数值题：确定性自动评分
  → 开放题：LLM-as-Judge + 一致性校验（两次裁判不一致则人工复核）


 防线5：异常检测
  → 模型响应速度异常（太快/超时）→ 记录并标注
  → 答案格式异常（拒绝回答/乱码）→ 记 0 分并记录
  → 整体表现偏差过大（>3σ）→ 人工复查后再发布

 评测任务隔离：每个模型评测在独立容器中运行，互不干扰
 失败重试：子任务失败自动重试 3 次（指数退避）
